<objective>
Add SQLite migration step 22 to store.rs: UPDATE decoded_frames to replace MAVERICK/PUFFIN
device_type rows with GOOSE, and bump CURRENT_SCHEMA_VERSION from 21 to 22.

Purpose: Normalises the device_type column so all historical rows use canonical values
(GEN_4, GOOSE, HR_MONITOR). This is idempotent — safe to run on any existing DB. After
migration, parse_device_type() in bridge.rs (Plan 03) will reject MAVERICK/PUFFIN, so
no new rows will ever carry those values (per D-09, D-10, D-11).

- CURRENT_SCHEMA_VERSION bumped to 22 (line 14 of store.rs)
- Migration SQL added to migrate() batch after the v21 INSERT
- Unit tests: idempotency test and row verification test
