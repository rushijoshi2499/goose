<objective>
Activate the `deny(clippy::unwrap_used)` lint at the crate root (scoped to non-test
code via `cfg_attr(not(test), ...)`), install per-module `#![allow(clippy::unwrap_used)]`
shields on every module that still has unconverted `.unwrap()` calls so the build stays
green, and convert all 46 test-code `.unwrap()` calls in `bridge.rs` to
`.expect("descriptive message")` (D-03).

This is the foundation plan: it turns the lint on from day one (D-04) and creates the
progressive allow-removal structure that Plans 2–5 unwind. bridge.rs has **zero**
production unwrap violations (confirmed by clippy on HEAD), so all bridge.rs work here is
the D-03 test-quality pass plus the lib.rs attribute and allow shields.

Purpose: Establish the lint gate and shield structure so the allow list visibly shrinks
with each subsequent plan, while keeping the crate compiling at every step.
Output: lib.rs with the deny attribute; bridge.rs test code using `.expect()`; file-level
`#![allow(clippy::unwrap_used)]` on store.rs, metrics.rs, capabilities.rs,
openwhoop_reference.rs, exercise_detection.rs, energy_rollup.rs, step_discovery.rs.
</objective>
