## Deferred Requirements

### CAPSENSE-01 — Hardware Gate

**Reason:** Cap sense GATT characteristic UUID cannot be determined from static binary analysis alone. The `WHPWhoopStrapCapSenseFailedNotification` / `WHPWhoopStrapCapSenseSuccessNotification` notification names are present in the binary but the UUID-to-notification mapping requires decompilation of CapSenseViewModel functions.

**Candidates for next investigation:**
- `11500005-6215-11EE-8C99-0242AC120002`
- `11500007-6215-11EE-8C99-0242AC120002`

**Unblock path:** Connect real WHOOP 5.x device and subscribe to each 11500X UUID; observe which correlates with on-wrist state changes. Or BLE sniffer during cap sense calibration flow.

**Next milestone:** CAPSENSE-01 carries forward to v10.0 if hardware becomes available.
