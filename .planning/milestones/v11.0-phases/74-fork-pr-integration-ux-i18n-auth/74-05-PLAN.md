<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Verify ChatGPT sign-in on physical device (PR-INT-05)</name>
  <what-built>
    Plan 74-04 replaced the non-standard auth.openai.com device auth paths with the standard RFC 8628 flow at auth0.openai.com. The device code endpoint is now /oauth/device/code; tokens are returned directly from /oauth/token poll; the custom intermediate authorization_code exchange step is removed.
  </what-built>
  <how-to-verify>
    This check requires a physical iOS device (not simulator) with a real ChatGPT account and network access.

    1. Build and install the app on device (or use existing TestFlight/AltStore build from this branch)
    2. Open Coach tab
    3. Tap the settings gear icon to open CoachSettingsSheet
    4. Tap "Sign in with ChatGPT" (or equivalent sign-in button)
    5. The app should display a user_code (e.g. "ABCD-EFGH") and a verification URL
    6. Open the verification URL on a browser (should be chatgpt.com/activate or similar)
    7. Enter the displayed user_code on the ChatGPT page and approve
    8. Return to the app — the sign-in flow should complete and show the account as connected (no error dialog)
    9. Verify the Coach tab now shows ChatGPT as the active provider

    If the flow fails:
    - If error is "invalid client" or "client_id not found": the client ID app_EMoamEEZ73f0CkXaXp7hrann may not be accepted by auth0.openai.com — document the exact error and return "sign-in-failed: [error message]"
    - If error is "access_denied" or similar after approval: the verification URL may have changed — document it
    - If the device code request itself fails with HTTP error: check network connectivity and confirm auth0.openai.com is reachable

    NOTE: If a fresh ChatGPT account is not available for testing, first call clearStoredAuth() via a debug action (if available) or uninstall/reinstall to start fresh.
  </how-to-verify>
  <resume-signal>Type "auth-ok" if sign-in completed successfully, or "auth-failed: [error description]" if the flow did not complete. If auth-failed, the team will investigate the client ID issue (Assumption A3 in RESEARCH.md).</resume-signal>
