# Phase 72 — UI Design Contract

> Visual and interaction contract for Phase 72: ANS tiles, Trends dashboard, and Manual Workout Entry sheet.
> Generated by gsd-ui-researcher. Source: 72-CONTEXT.md + codebase scan.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — native SwiftUI only |
| Preset | not applicable |
| Component library | SleepV2Palette + SleepV2StatCard + SleepV2Panel (existing, in-tree) |
| Icon library | SF Symbols (system images via `Image(systemName:)`) |
| Font | SF Pro (system default via `.font()` modifiers, `.fontDesign(.rounded)` for metric values) |

No external dependencies. No shadcn gate applies — this is a Swift/SwiftUI project with no npm/web tooling.

---

## Spacing Scale

All spacing values align with the 8-point grid already in use throughout `HealthRecoveryStressViews.swift` and `SleepV2InsightViews.swift`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text inline gap, badge padding |
| sm | 8px | Intra-row gaps (icon + label), caption gap |
| md | 14px | Card internal padding (matches `SleepV2StatCard.padding(14)`) |
| lg | 16px | Horizontal screen padding, panel padding |
| xl | 18px | `HealthRecoveryStressViews` content horizontal padding |
| 2xl | 24px | Section-to-section gap in scroll lists |
| 3xl | 34px | Bottom safe area buffer in scroll content |

Exceptions:
- ANS section header top padding: 24px (2xl) to visually separate from preceding Breakdown section.
- Sparkline card internal vertical padding: 16px top + 12px bottom (asymmetric — label sits at top, x-axis labels at bottom).
- Sheet form rows: 12px gap between label and control; 20px gap between form groups.
- Touch targets for effort stepper buttons: minimum 44px tap area (accessibility minimum).

---

## Typography

Derived from patterns in `SleepV2StatCard`, `StressV2BreakdownRow`, and `HealthDashboardMetricCard`. Four size roles used across all three UI surfaces in this phase.

| Role | SwiftUI Equivalent | Weight | Line Height | Usage |
|------|--------------------|--------|-------------|-------|
| Display | `.title3` + `.fontDesign(.rounded)` — ~20pt | semibold (600) | 1.2 | Metric values in ANS tiles and trend Y-axis endpoint labels |
| Heading | `.headline` — ~17pt | semibold (600) | 1.25 | Section headers ("ANS Balance", "7-Day Trends"), sheet title |
| Body | `.subheadline` — ~15pt | semibold (600) | 1.4 | Form field labels, picker row labels, sparkline card titles |
| Caption | `.caption` — ~12pt | semibold (600) | 1.3 | Metric unit suffix ("ms", "bpm"), x-axis date labels, secondary status copy |

Only two weight levels are used: **regular (400)** for `.foregroundStyle(palette.mutedText)` secondary captions, and **semibold (600)** for all other text. No bold (700) or heavy weights in this phase.

### 1. ANS Balance Section (extends `StressV2OverviewPage`)

Location: inserted into `StressV2OverviewPage.body`, inside the existing `VStack(alignment: .leading, spacing: 14)`, between the coaching card (`SleepV2CoachingCard`) and the "Timeline" section header.

#### Section header
- Use existing `SleepV2SectionHeader(title: "ANS Balance", palette: palette)`
- Top spacing: 24px above (matches spacing between other section headers)

#### ANS tile layout
```
HStack(spacing: 12) {
  ANSMetricTile(
    palette: palette,
    systemImage: "waveform.path.ecg",
    label: "HRV (RMSSD)",
    value: <formatted ms string>,
    unit: "ms",
    tint: palette.accent
  )
  ANSMetricTile(
    palette: palette,
    systemImage: "heart.fill",
    label: "Resting HR",
    value: <formatted bpm string>,
    unit: "bpm",
    tint: Color.red
  )
}
.frame(height: 96)
```

#### `ANSMetricTile` — new private struct, placed at bottom of `HealthRecoveryStressViews.swift`

Reuse `SleepV2StatCard` directly — it already accepts `systemImage`, `label`, and `value`. Compose the value string as `"\(numericValue) \(unit)"` before passing (e.g. `"42 ms"`, `"52 bpm"`). Do not create a new struct unless the value+unit layout requires two-line rendering.

Empty/unavailable state: pass `"No data"` as `value` (matches existing `SleepV2StatCard` pattern throughout the codebase).

Tile height: `.frame(height: 96)` — matches the two stat-card HStack above it.

Data bindings:
- HRV value: `HRVSeriesStore.shared.dailyEstimate()?.rmssdMS` — format as `"\(Int(v.rounded())) ms"` or `"No data"`
- RHR value: `HeartRateSeriesStore.shared.restingEstimate()` — format as `"\(Int(v.rounded())) bpm"` or `"No data"`

### 2. TrendsDashboardView — new screen

File: `GooseSwift/TrendsDashboardViews.swift`

Navigation: pushed via `NavigationStack` from Health tab (`HealthRoute.trends` case). Entry point: a `NavigationLink(value: HealthRoute.trends)` row added to `HealthDashboardViews.swift`, styled as a `CoachRouteRow`-equivalent inside an existing section.

#### Full layout structure

```
NavigationStack destination:
  ScrollView {
    VStack(alignment: .leading, spacing: 24) {
      // Section title (plain Text, not SleepV2SectionHeader — no palette in this view)
      Text("7-Day Trends")
        .font(.headline.weight(.semibold))
        .foregroundStyle(.primary)

      // Three sparkline cards, stacked vertically
      TrendsSparklineCard(metric: .recovery, data: recoveryPoints, tint: .green)
      TrendsSparklineCard(metric: .hrv,      data: hrvPoints,      tint: Color(red:0.33,green:0.72,blue:0.70))
      TrendsSparklineCard(metric: .strain,   data: strainPoints,   tint: .orange)
    }
    .padding(.horizontal, 16)
    .padding(.vertical, 16)
  }
  .gooseScreenBackground()
  .navigationTitle("Trends")
  .navigationBarTitleDisplayMode(.inline)
```

#### `TrendsSparklineCard` — new struct

Container: `VStack(alignment: .leading, spacing: 12)` with 16px all-around padding, `.quaternary.opacity(0.4)` background, `RoundedRectangle(cornerRadius: 14, style: .continuous)`.

Internal layout:

```
VStack(alignment: .leading, spacing: 12) {
  // Header row
  HStack(spacing: 8) {
    Image(systemName: metric.systemImage)           // 16pt semibold, tint color
      .frame(width: 28, height: 28)
      .background(tint.opacity(0.14), in: RoundedRectangle(cornerRadius: 8, style: .continuous))
    Text(metric.title)                              // .subheadline.weight(.semibold), .primary
    Spacer()
    Text(latestValueFormatted)                      // .title3.weight(.semibold), .fontDesign(.rounded), tint color
  }

  // Sparkline Path
  TrendsSparklineShape(points: normalizedPoints)
    .stroke(tint, style: StrokeStyle(lineWidth: 2.5, lineCap: .round, lineJoin: .round))
    .frame(height: 52)
    .background(alignment: .bottom) {
      tint.opacity(0.07)
        .clipShape(TrendsSparklineShape(points: normalizedPoints).fill())  // area fill
    }

  // X-axis date labels — first, middle, last of the 7-day window
  HStack {
    Text(dateLabels.first ?? "")
    Spacer()
    Text(dateLabels[3])
    Spacer()
    Text(dateLabels.last ?? "")
  }
  .font(.caption.weight(.semibold))
  .foregroundStyle(.secondary)
}
.padding(16)
.background(.quaternary.opacity(0.4), in: RoundedRectangle(cornerRadius: 14, style: .continuous))
```

Card height: unconstrained (intrinsic) — approximately 136px with the 52px sparkline.

#### Sparkline `Path` specification

`TrendsSparklineShape: Shape` — a `Path` polyline computed from normalized points.

Point normalization:
1. Input: array of `(date: Date, value: Double)` sorted ascending, 7 entries (fill missing days with `nil`; skip nil points in the path).
2. X axis: evenly spaced. `x = leftInset + (index / 6.0) * usableWidth`. `leftInset = 0`, `usableWidth = geometry.width`.
3. Y axis: inverted (SwiftUI origin top-left). `y = usableHeight - ((value - minValue) / (maxValue - minValue)) * usableHeight`. `usableHeight = geometry.size.height`. Clamp to `[0, usableHeight]`.
4. If `maxValue == minValue` (flat line): render at `y = usableHeight / 2`.
5. Path construction: `path.move(to: firstPoint)`, then `path.addLine(to: nextPoint)` for each subsequent non-nil point. No bezier smoothing — straight polyline segments (matches `StressV2TimelineChart` pattern).
6. Area fill shape (for background gradient): same path, then `path.addLine` to bottom-right corner, then `path.addLine` to bottom-left corner, then `path.closeSubpath()`.

Empty state (zero data points): show `Text("No data for the last 7 days").font(.caption).foregroundStyle(.secondary)` centered in the card's sparkline area instead of the `Path`.

### 3. ManualWorkoutEntrySheet — new `.sheet` modal

File: `GooseSwift/ManualWorkoutEntryViews.swift`

Presentation: `.sheet` modifier, `presentationDetents([.medium, .large])`. Presented from a "Log Workout" `Button` or toolbar item in the Fitness/Activity context (exact placement: toolbar trailing item in `FitnessView` or equivalent activity list view — at Claude's discretion per CONTEXT.md).

#### Full layout

```
NavigationStack {
  Form {
    // Section 1: Activity
    Section("Activity") {
      Picker("Sport", selection: $selectedKind) {
        ForEach(ActivityKind.allCases) { kind in
          Text(kind.displayName).tag(kind)
        }
      }
      .pickerStyle(.menu)
    }

    // Section 2: Duration
    Section("Duration") {
      HStack {
        Text("Minutes")
          .foregroundStyle(.primary)
        Spacer()
        // Stepper with numeric display
        Stepper("\(durationMinutes) min", value: $durationMinutes, in: 1...600, step: 5)
      }
    }

    // Section 3: Perceived Effort
    Section {
      VStack(alignment: .leading, spacing: 12) {
        HStack {
          Text("Perceived Effort")
            .font(.subheadline.weight(.semibold))
          Spacer()
          Text("\(effortValue) / 10")
            .font(.subheadline.weight(.semibold))
            .fontDesign(.rounded)
            .foregroundStyle(Color.orange)
        }
        // 10-segment effort selector (horizontal row of tappable dots/capsules)
        EffortScaleSelector(selectedValue: $effortValue)
      }
      .padding(.vertical, 4)
    } header: {
      Text("Effort")
    }
  }
  .navigationTitle("Log Workout")
  .navigationBarTitleDisplayMode(.inline)
  .toolbar {
    ToolbarItem(placement: .topBarLeading) {
      Button("Cancel") {
        dismiss()
      }
    }
    ToolbarItem(placement: .topBarTrailing) {
      Button("Log") {
        submitWorkout()
      }
      .fontWeight(.semibold)
      .foregroundStyle(isFormValid ? Color.orange : Color.secondary)
      .disabled(!isFormValid)
    }
  }
}
```

#### `EffortScaleSelector` — new private struct

Layout: `HStack(spacing: 6)` of 10 capsule buttons, each labeled 1–10.

```
ForEach(1...10, id: \.self) { value in
  Button {
    selectedValue = value
  } label: {
    Text("\(value)")
      .font(.caption.weight(.semibold))
      .frame(maxWidth: .infinity)
      .frame(height: 32)
      .background(
        selectedValue == value
          ? Color.orange
          : Color.orange.opacity(0.12),
        in: RoundedRectangle(cornerRadius: 8, style: .continuous)
      )
      .foregroundStyle(
        selectedValue == value ? Color.white : Color.orange
      )
  }
  .buttonStyle(.plain)
}
```

Minimum tap target: each capsule spans equal fraction of available width. With 16px horizontal padding on each side and 6px gaps, each button is approximately 28px wide — below the 44px minimum. Compensate by applying `.contentShape(Rectangle().size(width: 44, height: 44))` or wrapping in a `.frame(minWidth: 44, minHeight: 44)` clip group. On a standard 390pt screen this lays out at ~31pt each; acceptable with explicit `.contentShape` expansion.

#### Form defaults

| Field | Default | Range / Constraint |
|-------|---------|-------------------|
| Sport | `.run` | All `ActivityKind.allCases` |
| Duration | 30 min | 1–600 min, step 5 |
| Effort | 5 | 1–10, integer |

Validation: Log button enabled only when `durationMinutes >= 1`. Sport and effort always have valid defaults, so validation only guards duration.

#### Submission

On "Log" tap:
1. Disable Log button immediately (set `isSubmitting = true`).
2. Call `workout.upsert` bridge with `{ date: ISO8601 today, sport: selectedKind.rawValue, duration_s: durationMinutes * 60, perceived_effort: effortValue }`.
3. On success: `dismiss()`.
4. On error: show inline error text below the form (`.foregroundStyle(.red)`, `.caption` font). Log button re-enabled.

## Interaction States

### ANS tiles (in Stress view)

| State | Behavior |
|-------|----------|
| Loading | Tiles show `"—"` as value while data is being fetched (same `.onAppear` Task pattern as existing stress view) |
| Data available | Display formatted value + unit |
| No data | Display `"No data"` (string, same as existing stat cards) |
| No interaction | Tiles are non-tappable (display only, no NavigationLink) |

### Trends sparkline cards

| State | Behavior |
|-------|----------|
| Loading | Show `ProgressView()` at card height in place of sparkline |
| Data available | Render `Path` polyline with area fill |
| Partial data (< 7 days) | Render available points; x-axis still spans full 7-day range; missing days = gap in line |
| No data | Show `"No data for the last 7 days"` inline text |
| Cards are non-tappable in this phase (no drill-down per CONTEXT.md deferred section) |

### Manual Workout Entry sheet

| State | Behavior |
|-------|----------|
| Default | Sport = Run, Duration = 30 min, Effort = 5; Log button enabled |
| Duration = 0 | Log button disabled (gray, opacity reduced) |
| Submitting | Log button disabled, `ProgressView()` inline or button label replaced with `ProgressView()` |
| Success | Sheet dismissed |
| Error | Error string shown below form; Log button re-enabled |

## Accessibility

- All `SleepV2StatCard` instances (ANS tiles) use their `label` string as the accessibility label implicitly via SwiftUI text rendering.
- Effort scale buttons: each button must have `.accessibilityLabel("Effort \(value) of 10")`.
- Sparkline `Path` is decorative — apply `.accessibilityHidden(true)` and provide `.accessibilityLabel("Recovery trend over 7 days")` on the card container.
- Log button: `.accessibilityLabel("Log workout")`.
- Cancel button: `.accessibilityLabel("Cancel workout entry")`.
- Minimum touch target for stepper and effort capsules: 44pt height enforced via `.contentShape`.
