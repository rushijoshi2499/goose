<objective>
Add a `GooseBLEDataValidator` struct that gates structurally invalid BLE frames before bytes reach the Rust bridge. It is invoked in the notification ingest pipeline (`GooseAppModel+NotificationPipeline.swift`, inside `parseNotificationFrames` before `parser.parseBatch`). Invalid frames are logged via OSLog at warning level and counted in `invalidFrameCount` on GooseBLEClient, surfaced in More > Debug.

Validator invariants (structural ONLY — per CONTEXT decision and the BLE5-04 pitfall): device_id non-nil, payload non-nil, payload non-empty (length >= 1). NO packet-type whitelist — a whitelist would silently break WHOOP 5.0 support.

Purpose: Prevent malformed frames from wasting Rust-bridge round-trips and polluting SQLite, while never silently dropping unknown-but-valid packet types. Implements BLE5-04.

Output: New `GooseBLEDataValidator.swift`; `invalidFrameCount` + validator wiring on GooseBLEClient; validator call in the notification pipeline; a debug row in MoreDebugViews; pbxproj reference.
</objective>
