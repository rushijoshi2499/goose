# Phase 71 тАФ UI Design Contract

## Design System

| Property | Value |
|----------|-------|
| Tool | none тАФ SwiftUI native, no shadcn |
| Preset | not applicable |
| Component library | SwiftUI + UIKit (native iOS 26.0) |

**Note:** Project uses `FitnessColor` enum (defined in `FitnessFormatting.swift`) for session-screen tokens, and `coachCardSurface(tint:prominent:)` modifier for Coach tab cards. Both must be used where applicable in this phase.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, tight inline padding |
| sm | 8px | Button padding, inter-row gaps in metric list |
| md | 12тАУ16px | Card internal padding (12 for compact, 16 for standard) |
| lg | 18тАУ24px | Section vertical padding, VStack spacing in scroll views |
| xl | 32px | Bottom safe-area padding for session CTAs |
| touch | 44pt | Minimum tap target for all interactive controls |

Exceptions:
- Stepper controls in IntervalTimerView use 44pt minimum touch targets regardless of visual size
- VOW nudge card uses 12px internal padding to remain visually lighter than `CoachRecommendationCard` (16px)
- BreatheView precedent: CTA button is 160pt wide ├Ч 48pt tall with `.bottom` padding of 32pt тАФ IntervalTimerView must match this exactly

## Typography

All sizes use SF Pro (system font). Weights are limited to two: regular (400) and semibold (600).

| Role | SwiftUI Spec | Weight | Line Height |
|------|-------------|--------|-------------|
| Body | `.body` (~17pt) | regular (400) | 1.5 (system default) |
| Label / subheadline | `.subheadline` (~15pt) | semibold (600) for titles; regular for secondary |
| Caption | `.caption` (~12pt) / `.caption2` (~11pt) | semibold for values; regular for metadata |
| Display (countdown) | `.system(size: 64, weight: .semibold, design: .rounded)` | semibold (600) | 1.0 (single line) |
| Section header | `.system(size: 11, weight: .black)` | black (900) | тАФ |

Rules:
- Section headers (e.g. "WELLNESS", "DATA") use `.system(size: 11, weight: .black)` with `.foregroundStyle(.secondary)` тАФ matching the `DailyJournalSheet` "TAGS" header pattern exactly
- Countdown timer digits use `.fontDesign(.rounded)` to match `CoachMetricHighlightCard` value style
- Metric Explorer values use `.caption.weight(.semibold)` + `.fontDesign(.rounded)` тАФ same as highlight cards
- Tracking / letter-spacing: only applied to BreatheView phase labels (tracking 2.0); IntervalTimerView phase label inherits this pattern

## Color

This phase uses two distinct color contexts:

### Context A тАФ Coach Tab (VOW nudge card)

Follows the `coachCardSurface(tint:prominent:)` modifier pattern from `CoachView.swift`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `Color(.secondarySystemGroupedBackground)` | Card fill via `coachCardSurface` |
| Secondary (30%) | `Color(.systemGroupedBackground)` | Page scroll background (inherited from `CoachOverviewScreen`) |
| Accent тАФ Critical | `.red` | VOW card tint when recovery < 33% |
| Accent тАФ Warning | `.orange` | VOW card tint when recovery < 66% or strain > 18 |
| Accent тАФ Info | `.blue` | VOW card tint when HRV < 30ms |

Accent reserved for: VOW card tint color (border stroke + icon tint + shadow) only. Never applied to text.

### Context B тАФ Session Screens (IntervalTimerView, MetricExplorerView)

Follows `BreatheView` / `FitnessColor` dark-background pattern.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `FitnessColor.background` (`Color.black`) | Full-screen background, ignores safe area |
| Secondary (30%) | `FitnessColor.panel` (`rgb(0.10, 0.10, 0.11)`) | Card surfaces, stepper backgrounds, metric row backgrounds |
| Accent тАФ Active | `FitnessColor.standCyan` (`rgb(0.39, 0.92, 0.95)`) | Active state: Start button label, work-phase indicator, running countdown |
| Accent тАФ Rest | `FitnessColor.workoutYellow` (`rgb(1.0, 0.91, 0.24)`) | Rest-phase indicator in IntervalTimerView only |
| Secondary text | `FitnessColor.secondaryText` (`rgb(0.58, 0.58, 0.62)`) | Subtitles, timestamps, metadata |
| Stop / destructive | `FitnessColor.endRed` (`rgb(1.0, 0.25, 0.27)`) | Stop button label when session is active |

Accent reserved for: active-session state indicator (phase label color, countdown color), Start button foreground, metric value highlights. Not used for backgrounds except at 14% opacity for the Start button container.

### Semantic colors (both contexts)

| Signal | Color | Usage |
|--------|-------|-------|
| Critical (recovery < 33%) | `.red` | VOW card tint only |
| Warning (recovery < 66%, strain > 18) | `.orange` | VOW card tint only |
| Low HRV (< 30ms) | `.blue` | VOW card tint only |
| Destructive | `.red` / `FitnessColor.endRed` | Stop button label (session screens) |

## Component Specs

### 1 тАФ VOW Nudge Card (`CoachVOWCard`)

**Placement:** First item inside `CoachOverviewScreen`'s `LazyVStack`, above `CoachRecommendationCard`. Only rendered when a nudge is active. Dismissed by swipe-down gesture (session-scoped, no persistence).

**Layout:**

```
HStack(spacing: 12) {
  [Icon 28pt, tinted, 36├Ч36 rounded rect bg at 12% opacity]
  VStack(alignment: .leading, spacing: 4) {
    Text(urgencyLabel)     // .subheadline.weight(.semibold)
    Text(nudgeBody)        // .caption, .foregroundStyle(.secondary), max 2 lines
  }
  Spacer()
  [Dismiss button: Image(systemName: "xmark") at .caption2 weight .semibold, .tertiary]
}
.padding(12)
.coachCardSurface(tint: urgencyTint, prominent: false)
```

**Urgency mapping:**

| Condition | Label | Icon | Tint |
|-----------|-------|------|------|
| recovery < 33% | "Critical Recovery" | `heart.slash` | `.red` |
| recovery < 66% | "Low Recovery" | `heart` | `.orange` |
| strain > 18 | "High Strain" | `figure.run` | `.orange` |
| HRV < 30ms | "Low HRV" | `waveform.path.ecg` | `.blue` |

Priority order (highest wins, only 1 shown): Critical Recovery > Low Recovery > High Strain > Low HRV.

**Nudge body copy:**

| Condition | Body text |
|-----------|-----------|
| recovery < 33% | "Recovery is critically low. Prioritise rest and avoid high-strain activity today." |
| recovery < 66% | "Recovery is below 66%. Consider light training only." |
| strain > 18 | "Strain is high. Allow adequate recovery before the next session." |
| HRV < 30ms | "HRV is low this week. Monitor stress and sleep quality." |

**Dismiss interaction:**
- `@State private var isDismissed = false`
- Tap the `xmark` button тЖТ `isDismissed = true`
- Swipe down (`.gesture(DragGesture().onEnded { if $0.translation.height > 30 { isDismissed = true } })`) тЖТ same
- `isDismissed` is not persisted тАФ card reappears on next app launch / tab re-enter
- No animation on dismiss: use `.opacity(isDismissed ? 0 : 1)` with `.animation(.easeOut(duration: 0.2), value: isDismissed)`

**Accessibility:**
- `.accessibilityElement(children: .combine)`
- `.accessibilityLabel("\(urgencyLabel). \(nudgeBody). Double-tap to dismiss.")`

### 2 тАФ IntervalTimerView (full screen)

**Entry:** `MoreRoute.intervalTimer` тЖТ `NavigationLink` destination. Follows `BreatheView` structure exactly.

**Modifiers (must match BreatheView):**
- `.navigationTitle("Interval Timer")`
- `.navigationBarTitleDisplayMode(.inline)`
- `.toolbar(.hidden, for: .tabBar)`
- `.background(FitnessColor.background.ignoresSafeArea())`
- `.toolbarBackground(FitnessColor.background, for: .navigationBar)`
- `.toolbarColorScheme(.dark, for: .navigationBar)`
- `.onDisappear { stopSession() }`

**Layout (ZStack root):**

```
ZStack {
  FitnessColor.background.ignoresSafeArea()

  VStack(spacing: 0) {
    Spacer()

    // --- Config area (shown only when !isRunning) ---
    if !isRunning {
      VStack(spacing: 24) {
        IntervalStepperRow(label: "Work", seconds: $workSeconds)   // default 30s
        IntervalStepperRow(label: "Rest", seconds: $restSeconds)   // default 10s
      }
      .padding(.horizontal, 32)
    }

    // --- Countdown display (shown when isRunning) ---
    if isRunning {
      VStack(spacing: 8) {
        Text(currentPhaseLabel)          // "WORK" or "REST"
          .font(.system(size: 20, weight: .semibold))
          .tracking(2.0)
          .foregroundStyle(currentPhaseColor)
          .animation(.easeInOut(duration: 0.25), value: currentPhase)
          .contentTransition(.opacity)

        Text(countdownText)             // "00:28"
          .font(.system(size: 64, weight: .semibold, design: .rounded))
          .foregroundStyle(currentPhaseColor)
          .monospacedDigit()
          .contentTransition(.numericText(countsDown: true))
      }
    }

    Spacer()

    // --- BLE hint (when !isRunning and not connected) ---
    // Same pattern as BreatheView: sensor.tag.radiowaves.forward icon + text

    // --- CTA ---
    Button(isRunning ? "Stop" : "Start") { isRunning ? stopSession() : startSession() }
      .font(.body.weight(.semibold))
      .foregroundStyle(isRunning ? FitnessColor.endRed : FitnessColor.standCyan)
      .frame(width: 160, height: 48)
      .background(
        isRunning ? FitnessColor.endRed.opacity(0.14) : FitnessColor.standCyan.opacity(0.14),
        in: Capsule()
      )
      .padding(.bottom, 32)
  }
}
```

**`IntervalStepperRow` sub-component:**

```
HStack {
  Text(label)                          // .subheadline.weight(.semibold), .foregroundStyle(.primary)
  Spacer()
  HStack(spacing: 16) {
    Button("-") { decrement() }
    Text(formattedSeconds)             // e.g. "30s" тАФ .subheadline.weight(.semibold), .monospacedDigit()
    Button("+") { increment() }
  }
  .font(.title3.weight(.semibold))
  .foregroundStyle(FitnessColor.standCyan)
}
.padding(14)
.background(FitnessColor.panel, in: RoundedRectangle(cornerRadius: 10, style: .continuous))
```

Stepper step: 5 seconds. Min work: 5s. Max work: 300s. Min rest: 5s. Max rest: 120s.

**Phase state:**
- `enum IntervalPhase { case work, rest }`
- `currentPhaseColor`: `.work` тЖТ `FitnessColor.standCyan`; `.rest` тЖТ `FitnessColor.workoutYellow`
- `currentPhaseLabel`: `.work` тЖТ `"WORK"`; `.rest` тЖТ `"REST"`
- Countdown format: `MM:SS` via `String(format: "%02d:%02d", seconds / 60, seconds % 60)`

**Session loop pattern (matches BreatheView exactly):**
```swift
phaseTask = Task { @MainActor in
  repeat {
    currentPhase = .work
    model.ble.buzz(loops: 1)
    // countdown workSeconds тЖТ 0
    guard !Task.isCancelled else { break }
    currentPhase = .rest
    model.ble.buzz(loops: 1)
    // countdown restSeconds тЖТ 0
  } while !Task.isCancelled
}
```

**Accessibility:**
- Start button: `.accessibilityLabel("Start interval timer")`
- Stop button: `.accessibilityLabel("Stop interval timer")`
- Countdown: `.accessibilityLabel("\(currentPhaseLabel) тАФ \(countdownText) remaining")` + `.accessibilityAddTraits(.updatesFrequently)`

### 3 тАФ MetricExplorerView (full screen)

**Entry:** `MoreRoute.metricExplorer` тЖТ `NavigationLink` destination. Uses `List` + `.insetGrouped` style to match `MoreView` conventions (not the dark BreatheView pattern тАФ this is a data-browser screen, not a session screen).

**Modifiers:**
- `.navigationTitle("Metric Explorer")`
- `.navigationBarTitleDisplayMode(.inline)`
- `.listStyle(.insetGrouped)`
- `.gooseListBackground()` (existing modifier used in MoreView)
- `.toolbarBackground(.hidden, for: .navigationBar)`

**Section structure:**

Each section maps to a metric family:

| Section Header | Metrics |
|----------------|---------|
| "READINESS" | Readiness, Recovery |
| "STRAIN" | Day Strain, Workout Strain |
| "CARDIOVASCULAR" | HRV, RHR |
| "SLEEP" | Sleep Duration, Sleep Efficiency |
| "STRESS" | Stress Score |

**Metric row layout:**

```
HStack(spacing: 12) {
  Image(systemName: metric.systemImage)
    .font(.caption.weight(.bold))
    .foregroundStyle(metric.tint)
    .frame(width: 28, height: 28)
    .background(metric.tint.opacity(0.12), in: RoundedRectangle(cornerRadius: 6))

  VStack(alignment: .leading, spacing: 2) {
    Text(metric.name)                  // .subheadline тАФ primary label
    Text(metric.timestamp)             // .caption2, .foregroundStyle(.tertiary) тАФ "Updated 3h ago" or "тАФ"
  }

  Spacer()

  Text(metric.displayValue)           // .subheadline.weight(.semibold), .fontDesign(.rounded)
                                       // "тАФ" when no data
}
.padding(.vertical, 4)
```

**Empty state (no data at all):**
- Displayed as a single List row spanning all sections
- `ContentUnavailableView("No Metrics Yet", systemImage: "chart.bar.xaxis", description: Text("Sync your WHOOP band to populate metrics."))`

**Error state:** Not applicable тАФ `healthStore.snapshot(for:)` never throws; missing data renders as `"тАФ"` value with `"No data"` timestamp.

**Accessibility:**
- Each row: `.accessibilityElement(children: .combine)`
- `.accessibilityLabel("\(metric.name), \(metric.displayValue), \(metric.timestamp)")`

## Notification Payloads (informational тАФ no UI screen)

These are `UNUserNotificationContent` payloads. No new view is introduced. Listed here so executor has exact copy.

| Trigger | Title | Body |
|---------|-------|------|
| Sleep synced | "Sleep synced" | "{duration} тАФ HRV {hrv}ms, Recovery {recovery}%" |
| Workout detected | "Workout detected" | "{duration} ┬╖ Strain {strain}" |
| Battery low | "WHOOP battery low" | "Battery at {pct}%. Charge your device." |

Format rules:
- Duration: `"7h 23m"` format (hours + minutes)
- HRV: integer milliseconds, no decimals
- Recovery: integer percentage
- Strain: one decimal (e.g. `"14.2"`)
- Battery percentage: integer

`.badge` is not added (Claude's discretion тАФ avoids icon badge clutter for health data notifications).
