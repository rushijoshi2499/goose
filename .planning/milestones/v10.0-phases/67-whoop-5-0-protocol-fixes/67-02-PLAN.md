<objective>
Implement BLE5-02: decode WHOOP 5.0 v18 (packet_k 18) historical frames per second, persist their biometric fields into the existing SQLite tables, and fix two historical-sync timestamp bugs (stale device clock and EVENT type-48 epoch mishandling) so WHOOP 5.0 historical offload imports cleanly without duplicates.

Purpose: WHOOP 5.0 historical sync emits v18 frames that the parser currently collapses into a bare `NormalHistory` summary (HR-present marker only) — all per-second HR / RR / gravity / skin-temp / step data is discarded. Two timestamp bugs additionally corrupt or duplicate historical rows on devices with stale clocks.

Output: A `parse_v18_body` parser + `V18History` body-summary variant (18 split out of the shared `7 | 9 | 12 | 18` arm), a bridge.rs persistence arm feeding the existing biometric stream vecs, stale-clock 300s grid snapping + EVENT type-48 bypass in historical_sync.rs, and v18 round-trip tests.
</objective>
