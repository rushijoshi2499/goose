<objective>
Implement BLE5-01: parse the WHOOP 5.0 R22 realtime packet (type 0x10) in the Rust core so WHOOP 5.0 users receive non-zero realtime HR / HRV / recovery metrics. Today type 0x10 falls through `parse_payload` to `ParsedPayload::Raw` and is silently discarded.

Purpose: WHOOP 5.0 devices stream realtime biometric data as R22 (type 0x10) packets that the current parser does not recognise. Without this, the Health tab shows zero realtime metrics for every WHOOP 5.0 user.

Output: A new packet-type constant, a dedicated `parse_r22_payload` parser, an `R22Whoop5Hr` body-summary variant routed through the existing `DataPacket`/trusted-frames pipeline, R17/R22 same-second dedup with R22 priority, and round-trip tests.
</objective>
