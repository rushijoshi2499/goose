|-----------|----------|-----------|-------------|-----------------|
| T-69-01 | Tampering | metric_name field in MetricSeriesUpsertArgs | mitigate | Validate metric_name is non-empty and matches [a-z0-9._-]+ pattern in bridge function before store call; reject with GooseError on invalid format |
| T-69-02 | Tampering | behaviors_json field in JournalUpsertArgs | accept | SQLite has no column size limit; behaviors_json is stored verbatim. Validate non-empty in bridge function (rejects ""); accept large payloads as low-risk (local device only, no network exposure) |
| T-69-03 | Denial of Service | metric_series bulk inserts | accept | Table is local-only SQLite; no rate limiting needed for a single-user app on-device |
| T-69-SC | Tampering | npm/pip/cargo installs | accept | No new packages introduced this phase; zero dependency surface change |
</threat_model>
