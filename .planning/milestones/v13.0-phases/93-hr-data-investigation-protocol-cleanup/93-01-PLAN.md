<objective>
Fix WHOOP 5.0 fw 50.38.1.0 HR data path (BUG-HR-01): two-site change in metric_features.rs that
makes R22Whoop5Hr frames contribute HR metrics.

Purpose: R22 BLE frames are correctly stored in SQLite (parse_r22_payload parses them;
capture_correlation.rs labels them "r22_whoop5_hr") but metric extraction silently drops them because
(a) trusted_frames_for_summary_kinds is not called with "r22_whoop5_hr" and (b) heart_rate_plan_from_row
has no R22Whoop5Hr match arm. Both gaps must close together — fixing only one produces no improvement.

Output: metric_features.rs with two targeted changes + a passing test that exercises the R22 extraction path.
</objective>
