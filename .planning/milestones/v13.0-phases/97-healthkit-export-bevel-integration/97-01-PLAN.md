<objective>
Add three Rust bridge methods that the Swift HealthKit exporter will call to read data from SQLite:
- store.hr_samples_between — reads hr_samples rows in a time range, returns [{ts, bpm}]
- store.spo2_samples_between — reads spo2_samples rows, applies inline SpO2 conversion via biometrics::spo2_from_raw, returns [{ts, spo2_percent}] (skipping low-confidence samples)
- store.external_sleep_sessions_between — wraps existing GooseStore::external_sleep_sessions_between, returns sleep session rows

Purpose: HK-01, HK-03, HK-04 depend on these bridge methods for reading WHOOP data from SQLite to write to HealthKit. No batch export is possible without these read paths.

Output: Three new bridge methods registered in capture.rs and mod.rs, covered by integration tests.
</objective>
