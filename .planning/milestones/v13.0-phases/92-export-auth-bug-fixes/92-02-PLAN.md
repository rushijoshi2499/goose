<objective>
Fix two independent export defaults bugs: (1) runFullRawExport() silently overrides the user's includeRawBytes setting by setting it to true; (2) the sqlite family Toggle in the Data Families section is not disabled when the SQLite file is > 20 MB, allowing the user to enable a preset that would include the oversized database. Per D-08 and D-10.

Purpose: Ensure runFullRawExport() respects the user-configured includeRawBytes flag, and guard the OOM-risk database inclusion by disabling the sqlite Toggle when the database exceeds 20 MB.

Output:
- GooseSwift/MoreDataStore+Validation.swift — runFullRawExport removes the includeRawBytes = true override
- GooseSwift/MoreDataStore.swift — adds isDatabaseTooLarge computed property derived from fetchSQLiteDBSizeLabel threshold
- GooseSwift/MoreRawExportViews.swift — sqlite family Toggle row gains .disabled(store.isDatabaseTooLarge)
</objective>
