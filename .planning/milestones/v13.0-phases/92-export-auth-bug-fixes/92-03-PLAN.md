<objective>
Fix the WHOOP 5.0 auth stuck state: after 12 insufficientAuthentication retry cycles the app currently only updates a connectionState string. Users have no actionable recovery path. Add an authRetryCount counter that increments at each retry-exhausted path; when the count reaches 12, set authExhausted = true to surface a SwiftUI .alert offering Reconnect WHOOP (disconnect + forget) and Cancel (halt loop, keep device) actions. Per D-01, D-02, D-03, D-04.

Purpose: Users stuck in auth failure can recover by tapping "Reconnect WHOOP", which clears the remembered device and returns them to the scan flow. The alert fires after 12 exhausted retry cycles — not after the first or second cycle. The retry loop is halted immediately when authExhausted is set.

Output:
- GooseSwift/CoreBluetoothBLETransport.swift — new var authRetryCount: Int = 0 and var authExhausted: Bool = false observable state
- GooseSwift/CoreBluetoothBLETransport+PeripheralDelegate.swift — increments authRetryCount at both retry-exhausted paths; sets authExhausted = true when count >= 12; resets both on successful write
- GooseSwift/CoreBluetoothBLETransport+CentralDelegate.swift — resets authRetryCount = 0 and authExhausted = false on disconnect
- GooseSwift/ConnectionView.swift — .alert bound to ble.authExhausted with Reconnect WHOOP / Cancel actions
</objective>
