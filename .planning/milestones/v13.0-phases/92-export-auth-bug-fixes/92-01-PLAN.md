<objective>
Fix two export OOM root causes: (1) the in-memory validation manifest dict is passed by value through multiple bridge calls instead of being released after disk write; (2) validate() is called twice inside createBundle() — the second call is redundant and computes the same result. Per D-05, D-06, D-07, D-09.

Purpose: Prevent iOS from OOM-killing the app on databases > 100 MB during export by releasing the large manifest dict after it is written to disk, then passing only the file path string to downstream bridge calls. Remove the dead second validate() call from createBundle().

Output:
- GooseSwift/MoreDataStore.swift — performRawExport uses manifestURL.path for runbook and review calls
- GooseSwift/GooseLocalDataExporter.swift — createBundle() hoists validate() before the do block; result reused in both paths; second call removed
- Rust/core/src/bridge/debug.rs — LocalHealthValidationManifestRunbookArgs and LocalHealthValidationManifestReviewArgs accept manifest_path; bridges read JSON from disk
</objective>
