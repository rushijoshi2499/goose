<objective>
Update the Swift BLE transport layer to detect WHOOP MG at advertisement time, pass "WHOOP_MG" to the Rust bridge, and display "WHOOP MG" in the device view (D-06).

Purpose: Closes MG-02 — MG devices currently fall through to WHOOP5 identification because the detection logic has only two branches (gen4 / gen5). This plan adds the third branch using peripheral advertised name as the disambiguator (service UUID alone cannot distinguish MG from Goose — both use fd4b0001).
Output: Three Swift files updated; MG devices show "Gen MG" or equivalent label per D-06 in DeviceConnectionHeader.
</objective>
