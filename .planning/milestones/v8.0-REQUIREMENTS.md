## v1 Requirements (Archived)

### Bug Audit

- [x] **AUDIT-01**: Code review de v6.0–v7.0 (fases 36–50) identifica e corrige bugs de correctness, data races e edge cases em Rust e Swift
  - *Outcome: 3 HIGH + 6 MEDIUM bugs fixed; data race on GooseRustBridge eliminated; fabricated values removed.*

### Quick Tasks

- [x] **QT-01**: O utilizador consegue abrir as definições de Bluetooth do iOS directamente a partir do botão BT na app
  - *Outcome: DeviceView "BT Settings" button opens iOS Bluetooth Settings.*
- [x] **QT-02**: O CodeQL corre automaticamente em PRs/pushes via GitHub Actions
  - *Outcome: CodeQL CI already confirmed in .github/workflows/codeql.yml.*
- [x] **QT-03**: O utilizador consegue importar dados do HealthKit via HealthKit Full Importer
  - *Outcome: HealthKit importer confirmed in MoreView + HealthKitFullImporter.*

### Home — Surfaces

- [x] **HOME-01**: HomeDashboardView mostra Device Status Card com nome do device, estado da ligação, battery %, live HR, última sincronização e acção de reconnect quando desligado
  - *Outcome: HomeDeviceStatusCard added with live name, state, battery %, HR, last sync, reconnect action.*
- [x] **HOME-02**: HomeDashboardView mostra Tools Grid com atalhos para Sleep Coach, Activity, Journal e Calibration — cada um reflecte o estado de readiness do bridge
  - *Outcome: HomeToolsGrid with Sleep Coach, Activity, Journal, Calibration and readiness state.*
- [x] **HOME-03**: HomeDashboardView mostra Evidence Footer com versão do Rust core, caminho da store, modo de dados e provenance por família de métricas
  - *Outcome: HomeEvidenceFooter with Rust core version, store filename, data mode, algorithm family map; tap → More > Debug.*

### Coach — Routes e Conteúdo

- [x] **COACH-07**: CoachSummaries implementa score summaries para sleep, recovery, strain e stress
  - *Outcome: CoachMetricHighlightCard grid (sleep/recovery/strain/stress) from live bridge data.*
- [x] **COACH-08**: O utilizador consegue registar uma entrada de journal diária com prompt automático, tags opcionais e nota de texto persistida localmente
  - *Outcome: DailyJournalEntry + DailyJournalStore (UserDefaults) + DailyJournalSheet (TextEditor + tag chips).*
- [x] **COACH-09**: Sleep Coach route mostra wind-down time, target bedtime, wake time e sleep debt/fulfillment
  - *Outcome: CoachSleepRouteView with wind-down, bedtime, duration, debt.*
- [x] **COACH-10**: Recovery Insights route mostra recovery score, HRV, RHR, respiratory rate, skin temp delta e recomendação determinística
  - *Outcome: CoachRecoveryRouteView with score, z-HRV, z-RHR, HK biometrics, recommendation.*
- [x] **COACH-11**: Strain Guidance route mostra strain score, target strain, exercício, daytime HR e orientação under/in/over-target
  - *Outcome: CoachStrainRouteView with score, target, sessions, guidance.*
- [x] **COACH-12**: Stress Guidance route mostra stress score, último HRV/HR, breakdown por nível e non-activity stress quando disponível
  - *Outcome: CoachStressRouteView with score, avg HR, zones, non-activity note.*

### Biometrics — Recovery Fix

- [x] **BIO-05**: Recovery score usa z_rhr calculado de SpO2/resp/wrist temp reais dos pacotes V24 — elimina o baseline fabricado de 55.0 bpm
  - *Outcome: RHR baseline uses real 7-night history or neutral (70.0) — 55.0 eliminated.*

### Activity

- [x] **ACT-01**: Stress windows são particionadas por limites de sessões de exercício para que o non-activity stress só use amostras de HR fora de exercício
  - *Outcome: stressAlgorithmSummary filters out HR samples within exerciseSessions windows.*

### Persistência

- [x] **ENB-01**: Daily stress windows e estado do Energy Bank são persistidos em SQLite com rows diárias para histórico de longo prazo
  - *Outcome: persistEnergyDailyIfNeeded() calls metrics.energy_daily_rollup with write_metric=true.*

### Calibração

- [x] **CAL-01**: O utilizador consegue correr uma calibration pipeline real com splits train/holdout a partir de métricas locais históricas — resultados não hardcoded
  - *Outcome: calibrate() calls calibration.evaluate_stored_labels; calibrationSummary() shows real train/holdout counts.*

### Surface Cleanup

- [x] **SURF-01**: `previewMissingData` e strings de preview só são avaliados em `#if DEBUG` — nenhum valor fabricado visível ao utilizador em builds de produção
  - *Outcome: applyPreviewState + call site wrapped in #if DEBUG.*

### More Tab

- [x] **MORE-01**: More tab completa as actions de capture imports, backfill fields, raw export com famílias seleccionáveis, e links de privacy (eliminação e export de dados)
  - *Outcome: MorePrivacyView with ShareLink for SQLite export; destructive confirmation for local data deletion.*

### Previews

- [x] **PREV-01**: Previews SwiftUI existem para HomeDashboardView, CoachView e More views — cada um com estados connected/populated, disconnected e no-data — verificados com simulator screenshot
  - *Outcome: #Preview macros for Home (disconnected + populated) and More views.*

### Health

- [x] **HALG-01**: HealthDataStore implementa `algorithmPreferences` e `referenceAlgorithmDefinitions` wired ao bridge catalog
  - *Outcome: algorithmPreferences and referenceAlgorithmDefinitions shims wired to bridge catalog.*

### Band Sleep

- [x] **BAND-01**: Sleep records são ingeridos directamente de pacotes BLE (band sleep import path) — UI deixa de mostrar "band sleep import not available"
  - *Outcome: HealthDashboardViews shows bandSleepImportStatus; band sleep import pipeline was already complete since Phase 50.*

- ALG-HRV-04 / VAL-HRV-01: RMSSD cross-validated em ≥5 sessões overnight reais — hardware gate

## Out of Scope

- PRs de volta ao upstream b-nnett/goose com fixes do fork
- Server-side data analysis (dashboards web, alertas)
- Advanced authentication (OAuth, 2FA) — Bearer token é suficiente
- Full Android app — fundações de arquitectura em v2.0 são suficientes
- Offline mode — real-time é o core value
