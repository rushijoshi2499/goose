# Phase 110 Plan 02: r2d2 SQLite Connection Pool — Bridge Migration

## Objective

Wire the already-declared `r2d2` and `r2d2_sqlite` dependencies (in Cargo.toml since v13.0) into the bridge dispatcher. Replace per-request `GooseStore::open()` / `acquire_bridge_conn()` calls with a process-lifetime `r2d2::Pool` checkout. `GooseStore` struct and all test code remain unchanged — pool is bridge-layer only.

- Bridge domain files use `acquire_bridge_conn` (majority) or `open_bridge_store_hot` (a few) per handler call

    4. Add pool initializer function (after `acquire_bridge_conn`):
       ```rust
       fn init_bridge_pool(database_path: &str) -> GooseResult<BridgePool> {
           validate_no_traversal("database_path", database_path)?;
           // Run migration once before handing the path to the pool manager
           let migration_path = Path::new(database_path);
           GooseStore::open(migration_path)?; // runs migrate(), then drops the connection
           // Build pool — SqliteConnectionManager opens connections on demand
           let manager = SqliteConnectionManager::file(database_path)
               .with_flags(
                   rusqlite::OpenFlags::SQLITE_OPEN_READ_WRITE
                       | rusqlite::OpenFlags::SQLITE_OPEN_CREATE,
               )
               .with_init(|conn| {
                   conn.execute_batch("PRAGMA journal_mode=WAL; PRAGMA foreign_keys=ON;")?;
                   Ok(())
               });
           r2d2::Pool::builder()
               .max_size(4)
               .build(manager)
               .map_err(|e| GooseError::message(&format!("r2d2 pool init: {e}")))
       }
       ```

    5. Add checkout function (the new replacement for `acquire_bridge_conn`):
       ```rust
       /// Acquire a pooled SQLite connection for `database_path`.
       /// On first call for a given process lifetime, initialises the pool and runs schema
       /// migration. Subsequent calls return a checked-out connection from the pool.
       /// This replaces `acquire_bridge_conn` and `open_bridge_store_hot` at bridge handler
       /// call sites.
       pub(crate) fn checkout_bridge_conn(database_path: &str) -> GooseResult<BridgePoolConn> {
           if database_path.trim().is_empty() {
               return Err(GooseError::message("database_path is required"));
           }
           let pool = BRIDGE_CONN_POOL
               .get_or_try_init(|| init_bridge_pool(database_path))
               .map_err(|e| GooseError::message(&format!("pool init: {e}")))?;
           pool.get()
               .map_err(|e| GooseError::message(&format!("pool checkout: {e}")))
       }
       ```

    Keep `acquire_bridge_conn`, `open_bridge_store_hot`, `open_bridge_store`, and `BRIDGE_MIGRATED_PATHS` in place — they remain for backward compatibility with any call sites not yet migrated and for tests.
  </action>
