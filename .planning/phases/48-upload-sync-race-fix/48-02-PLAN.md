<objective>
Fix the race condition in GooseUploadService.performUpload (per D-01, D-02, D-03, D-04) and add a URLSession-injectable initialiser for testability (per D-05).

The fix is surgical — 2 production changes + 1 testability change:
1. Add `captureAllPendingRowIDs(deviceID:sinceTimestamp:) -> [String: [Int]]` — queries `sync.rows_pending_upload` for all 8 upload streams BEFORE `upload.get_recent_decoded_streams` is called.
2. Replace `markHrSamplesSynced(deviceID:sinceTimestamp:)` with `markStreamsSynced(rowIDsByStream:[String:[Int]])` — accepts pre-captured rowIDs, no internal re-fetch.
3. Add `init(databasePath:session:)` — takes an explicit URLSession for test injection.

Purpose: Eliminates blind-marking. Rows are only marked synced after the server confirms 2xx receipt. On 5xx or timeout, `markStreamsSynced` is never called and rows remain `synced=0` for the next cycle.

Output: Modified `GooseSwift/GooseUploadService.swift`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Swift → Rust bridge | sync.rows_pending_upload and sync.mark_synced receive stream names from Swift; stream names are validated by STREAM_ALLOWLIST in store.rs |
| Swift → URLSession | HTTP request body is JSON-serialized from Rust bridge output; no user-controlled input reaches the upload payload |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-48-02 | Tampering | stream name in sync.rows_pending_upload call | accept | Stream names are hardcoded constants in captureAllPendingRowIDs — not user-controlled; Rust STREAM_ALLOWLIST is the second line of defence |
| T-48-03 | Tampering | rowID values passed to sync.mark_synced | accept | rowIDs are integers captured from the local SQLite database, never from user input or server response |
| T-48-04 | Denial of Service | captureAllPendingRowIDs adds 8 extra bridge calls per upload | accept | All 8 calls are synchronous Rust reads on the detached utility task — no main thread impact; existing upload already makes similar calls |
| T-48-SC | Tampering | npm/pip/cargo installs | accept | No new packages — pure code refactor within existing stack |
</threat_model>
