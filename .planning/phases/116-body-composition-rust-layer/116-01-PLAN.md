## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Swift → bridge JSON | Untrusted args (`source`, `date`, metric values) cross from the app into Rust via `goose_bridge_handle_json` |
| bridge → SQLite | Bridge args become parameterized SQL bound values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-116-01 | Tampering | `source` arg → body_composition_history | mitigate | SQLite `CHECK(source IN ('manual','healthkit','scale'))` rejects invalid source; rusqlite error propagates as a bridge error response (no Rust re-validation needed) |
| T-116-02 | Tampering | SQL injection via date/source args | mitigate | All values bound via `rusqlite::params!` — no string interpolation into SQL |
| T-116-03 | Information disclosure | history_between returns all sources | accept | Local on-device SQLite only; no network surface in this phase; Swift filters per D-01 |
| T-116-04 | Denial of service | Unbounded date range query | accept | Single-user local DB, low row volume; acceptable; future pagination out of scope |
</threat_model>
