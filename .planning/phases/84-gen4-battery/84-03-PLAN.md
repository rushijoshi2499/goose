<objective>
Implement the eager Gen4 battery reading (BAT-02). On Gen4 connection, after `connectedCapabilities` is set and the link is ready, automatically send Cmd 26 (GET_BATTERY_LEVEL, command number 26) per locked decision D-01. Route the command-response frame to a new side-channel handler (mirroring `handleClockValue`/`handleClockCommandResponse`) that parses the payload via the `battery.parse_cmd26_response` bridge method (plan 84-01) on a background queue and publishes the result with `applyBatteryLevel`. This is the *initial* reading; the Event-48 path (plan 84-02) is the *live* reading that overrides it — no separate fallback state machine (D-04).

Purpose: Gives the user an immediate battery percentage on connect instead of waiting for a passive Event-48.
Output: A battery command enum case, an auto-send helper, a response handler, and the routing hook — all confined to GooseBLEClient files (no overlap with plan 84-02).
</objective>
