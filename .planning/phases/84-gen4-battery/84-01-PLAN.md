Add all Gen4 battery byte-parsing to the Rust core (per locked decision D-02: all parsing in Rust, never Swift). Two parsing paths — Event-48 (BAT-01) and Cmd 26 response (BAT-02) — are exposed as bridge methods AND covered by `#[cfg(test)]` unit tests (satisfies success criterion SC3). Purpose: Gen4 devices currently show no real battery percentage. This plan is the parsing foundation both Swift plans (84-02, 84-03) depend on.
Output: Two bridge methods, one new compact-summary field, and unit tests, all in Rust.

- Bridge method string `"battery.parse_event48_payload"` — args `{ "payload_hex": String }` (full Event-48 payload hex, header included), returns `{ "battery_pct": u16 }` or error.

- Private Rust fn `parse_cmd26_battery(payload: &[u8]) -> GooseResult<u16>` in bridge.rs (payload[2..4] u16 LE / 10, guard len >= 4).
- `pub(crate) fn read_u16_le` in protocol.rs (visibility widened from private).
