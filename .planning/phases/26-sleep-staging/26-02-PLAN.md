<objective>
Extend the Cole-Kripke binary spine (Plan 26-01) into the full 4-class sleep hypnogram (ALG-SLP-04):
classify each sleep epoch into wake/light/deep/rem from HR + motion threshold features, apply
physiological reimposition (no early REM, minimum 5-minute segment merge), derive AASM summary
metrics (TST, sleep efficiency, SOL, WASO, stage_minutes per class), and add the human
cross-validation gate (>= 5 sessions at >= 70% agreement) following the ALG-HRV-04 manual-gate pattern.

Purpose: A binary wake/sleep trace is not a usable hypnogram. WHOOP shows stage breakdown; this plan
delivers the 4-class output and the AASM metrics the Sleep V2 dashboard consumes. Because EEG-free
staging has a known literature ceiling (65-73%) and there are no proprietary WHOOP coefficients
(Ghidra-confirmed server-side, see 26-CONTEXT.md), the accuracy claim is validated by a human gate,
exactly as ALG-HRV-04 was.

Output: extended `sleep_staging.rs` (4-class classifier + reimposition + AASM + manual-gate doc
comment), updated `metrics.sleep_staging` bridge to accept HR feature inputs and return AASM metrics,
and a checkpoint that holds the phase open until the validation evidence is recorded.
</objective>

  <action>
    Extend `Rust/core/src/sleep_staging.rs`.
    (1) Per-epoch HR/motion feature input: add an optional per-epoch HR feature stream so the classifier can layer cardiorespiratory features on the binary spine. Add `EpochHrFeature { ts: f64, hr_bpm: f64 }` and accept `hr_features: &[EpochHrFeature]` in a new entry `pub fn stage_sleep_four_class(input: &SleepStagingInput, rows: &[(f64,f64,f64,f64)], hr_features: &[EpochHrFeature]) -> SleepStagingOutput`. Internally call the existing `stage_sleep` to get the binary wake/sleep spine, then refine each `sleep` epoch into light/deep/rem. Per ROADMAP #3: a sleep epoch with low relative HR (HR below the session p25 personal percentile) AND low motion (activity_count below a stillness threshold) -> "deep"; a sleep epoch in the later part of the night (clock proxy > 0.4, i.e. fractional position in the sleep period) with higher HR variability -> "rem"; remaining sleep epochs -> "light". Wake epochs stay "wake". Expose the thresholds as named constants (e.g. `DEEP_HR_PERCENTILE: f64 = 0.25`, `REM_CLOCK_PROXY_MIN: f64 = 0.4`, `DEEP_STILLNESS_ACTIVITY_MAX: f64 = ...`) — no magic literals at call sites. When `hr_features` is empty, fall back to the binary spine mapping sleep -> "light" (so the function never panics and always returns a valid 4-class trace).
    (2) Physiological reimposition, applied in order after per-epoch classification: (a) no REM in the first 15 minutes of sleep — reclassify any "rem" epoch whose offset from sleep onset < 15 min to "light"; (b) minimum 5-min segment merge — any contiguous run of the same class shorter than ceil(5.0 / COLE_KRIPKE_EPOCH_MINUTES) epochs is absorbed into the longer adjacent neighbour's class. Expose `MIN_SEGMENT_MINUTES: f64 = 5.0` and `NO_REM_ONSET_MINUTES: f64 = 15.0`. (Deep-concentration and forbidden-transition bridging from ROADMAP #4 are optional refinements per 26-CONTEXT <deferred> "if time permits" — implement rules (a) and (b), which are the binding minimum; if implementing deep→REM bridging, add it as a clearly named helper, do NOT reduce rules (a)/(b) to a "v1".)
    (3) AASM metrics: add fields to `SleepStagingOutput` (additive, keep existing fields): `tst_minutes: f64` (total sleep time = non-wake epochs * epoch minutes), `time_in_bed_minutes: f64`, `sleep_efficiency_fraction: f64` (tst/tib), `sol_minutes: f64` (sleep-onset latency: minutes from window start to first sleep epoch), `waso_minutes: f64` (wake epochs after first sleep onset * epoch minutes), `stage_minutes: BTreeMap<String, f64>` (minutes per class). Compute these from the final reimposed hypnogram in a helper `aasm_metrics(epochs, epoch_minutes)`.
    (4) `staging_method` continues to be `STAGING_METHOD_ACTIGRAPHY` for non-empty input and `STAGING_METHOD_NO_IMU` for empty input — the uncalibrated flag stays mandatory; shipping without it is a blocker.
    Add `#[cfg(test)] mod tests` cases: 4-class assignment on a synthetic still+low-HR window -> "deep"; late-night higher-HR window -> "rem"; reimposition rule (a) removes a REM epoch placed at minute 5; reimposition rule (b) merges a 2-epoch island into its neighbour; AASM derivation on a synthetic hypnogram (known TST/SOL/WASO/efficiency); staging_method == "actigraphy_uncalibrated" present on every non-empty 4-class output.
  </action>
