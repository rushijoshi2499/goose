# Phase 102 Plan 01: Gen4 skin_temp decode + MetricFeatures wire + issue #21 close

## Objective

Wire `skin_temperature_delta_c` from Gen4 V24 historical packet bytes into the MetricFeatures
pipeline. Verify `respiratory_rate` and RR interval paths already flow correctly. Close GitHub
issue #21 with a neutral summary comment. Run full Rust test suite to confirm no regressions.

## Context

All fields are already parsed by `parse_v24_body_summary` in `protocol.rs`. The store layer
already has `skin_temperature_delta_c` and `respiratory_rate_rpm` columns in
`daily_recovery_metrics`. The `backfill_streams_from_decoded_frames` in `store/capture.rs` already
inserts V24 RR intervals into the `rr_intervals` table (lines 981–998), satisfying D-03.

The single missing piece is `skin_temperature_plan_from_payload` in `metric_features.rs`: its
body_summary guard currently accepts `NormalHistory | V18History` only — `V24History` is excluded,
so skin temp from Gen4 frames is silently discarded. Adding `V24History` to the guard and a
`packet_k=24` arm with the NTC formula closes this gap.

`respiratory_rate_plan_from_payload` already includes `V24History` (added in Phase 99/GEN4-06)
with `raw_absolute_offset: 76`. No action needed there.
