# Phase 106-01: Android Metrics + Server Upload (AND-04)

## Objective

Wire up three features to the existing Android scaffold:
- D-04: Home tab shows live HR from `WhoopBleClient` `liveHeartRateBPM: StateFlow<Int?>`
- D-05: Health tab shows Recovery, Strain, Sleep scores via `GooseBridge` `metrics.*` bridge calls
- D-06: iOS v13.0 metric parity — same bridge method names as `HealthDataStore+Snapshots.swift`
- D-01: Server URL stored in Jetpack DataStore (Preferences), key `server_url`
- D-02: Upload triggered fire-and-forget on `Dispatchers.IO` via `completeSyncIfActive()` `onSyncComplete` callback
- D-03: POST to `{server_url}/v1/ingest-frames` using `HttpURLConnection`; skip silently when URL is empty

Also adds server URL settings UI (More tab) persisted via Jetpack DataStore. `./gradlew assembleDebug` must BUILD SUCCESSFUL.

- Package: `com.goose.app.upload`
- `object GooseUploadClient`
- `fun upload(context: Context, serverUrl: String)`:
  - If `serverUrl.isEmpty()`, return immediately (upload disabled)
  - Get db path: `context.filesDir.absolutePath + "/goose.sqlite"`
  - Call `GooseBridge.safeHandle()` with method `upload.get_recent_decoded_streams` and args `{"database_path": dbPath}` to get pending frames
  - Parse the response JSON; if `ok: false` or result is empty/null, return
  - POST the result payload to `"$serverUrl/v1/ingest-frames"` using `HttpURLConnection`:
    - `requestMethod = "POST"`
    - `setRequestProperty("Content-Type", "application/json")`
    - `doOutput = true`
    - Write payload bytes to `outputStream`
    - Read `responseCode`; log success (2xx) or failure
  - Catch all exceptions, log them — never throw to caller
- Use `android.util.Log` for logging (tag: `"GooseUpload"`)
- No external HTTP library — `java.net.HttpURLConnection` only
</action>
