<objective>
Eliminate the body_hex field duplication in cached parsed-payload JSON for K10/K21 raw-motion frames (PERF-05). K10 frames are ~1288 bytes and K21 ~1038 bytes; the body_hex hex dump roughly doubles the stored JSON for these high-volume motion frames while the structured body_summary (RawMotionK10/K21 axes) already carries the useful data.

Test-first is mandatory (per CONTEXT.md and the phase success criterion): first ADD body_hex assertions to the K10 and K21 tests that pin the CURRENT (populated) behaviour and confirm cargo test is green; then apply the exclusion in parse_data_packet_payload and UPDATE those same assertions to expect an empty body_hex; confirm cargo test is green again.

Purpose: shrink stored parsed-payload JSON for the highest-volume frame types before the IMU work in Phase 21 starts producing them at scale. Surgical: one conditional in one function, plus test assertions.
Output: updated protocol_tests.rs (K10/K21 body_hex assertions) and a conditional body_hex in protocol.rs parse_data_packet_payload.
</objective>
