Define `BodyCompositionRow` as a plain Swift struct (not Codable — parse from [String: Any] dict):

```
struct BodyCompositionRow {
  let date: String          // "yyyy-MM-dd"
  let source: String        // "manual" | "healthkit"
  let weightKg: Double?
  let bodyFatPct: Double?
  let muscleMassKg: Double?

  init?(from dict: [String: Any]) {
    guard let date = dict["date"] as? String else { return nil }
    self.date = date
    self.source = dict["source"] as? String ?? "manual"
    self.weightKg = dict["weight_kg"] as? Double
    self.bodyFatPct = dict["body_fat_pct"] as? Double
    self.muscleMassKg = dict["muscle_mass_kg"] as? Double
  }
}
```

Define the extension `extension HealthDataStore` (note: NOT @MainActor — the methods are already on a @MainActor class):

**loadBodyCompositionHistory():**
- Use `Self.hkDateFormatter` (nonisolated static let already present in HealthDataStore+Sleep.swift, format "yyyy-MM-dd")
- end = hkDateFormatter.string(from: Date()), start = hkDateFormatter.string(from: Date().addingTimeInterval(-7 * 24 * 60 * 60))
- Call `try await bridge.requestAsync(method: "body_composition.history_between", args: ["database_path": databasePath, "start_date": start, "end_date": end])`
- The bridge.requestAsync returns Any. Cast the result directly as `[[String: Any]]` (bare array — NOT wrapped in a "rows" key). If the direct cast fails, the history is empty: `self.bodyCompositionHistory = []` and return.
- Parse each element with BodyCompositionRow(from:), compact map, assign to self.bodyCompositionHistory.
- On catch: leave bodyCompositionHistory unchanged (do not clear existing data).

**upsertBodyComposition(date: String, weightKg: Double?, bodyFatPct: Double?, muscleMassKg: Double?, source: String):**
- Build args dict: always include "database_path", "date", "source". Include "weight_kg" only when non-nil, "body_fat_pct" only when non-nil, "muscle_mass_kg" only when non-nil.
- Call `try await bridge.requestAsync(method: "body_composition.upsert", args: args)`
- After success, call `await loadBodyCompositionHistory()` to refresh the stored property.
- On catch: do not crash; the caller (entry sheet) handles error state.

**importBodyCompositionFromHealthKit():**
- Guard `HKHealthStore.isHealthDataAvailable()` — return early if false (HealthKit unavailable, e.g., iPad without iPhone).
- Build a Set<HKObjectType> containing HKQuantityType.quantityType(forIdentifier: .bodyMass) and HKQuantityType.quantityType(forIdentifier: .bodyFatPercentage), guarding each optional with `if let`.
- Call `store.requestAuthorization(toShare: [], read: types)` wrapped in withCheckedThrowingContinuation (exact pattern from HealthKitFullImporter.requestAuthorization — read that file for the continuation pattern). On denial, return without crashing.
- Query HKSampleType.quantityType(forIdentifier: .bodyMass): use HKSampleQuery with NSSortDescriptor(key: HKSampleSortIdentifierStartDate, ascending: false), limit HKObjectQueryNoLimit. For each HKQuantitySample, extract start date (formatted yyyy-MM-dd via hkDateFormatter), quantity in kg (unit: HKUnit.gramUnit(with: .kilo)). Call await upsertBodyComposition(date:, weightKg:, bodyFatPct: nil, muscleMassKg: nil, source: "healthkit").
- Query HKQuantityType for bodyFatPercentage similarly: extract as percent() unit. Call await upsertBodyComposition(date:, weightKg: nil, bodyFatPct: value * 100.0, muscleMassKg: nil, source: "healthkit"). Note: HealthKit stores BF% as 0–1 fraction; multiply by 100 before passing to bridge.
- After both query loops complete, call await loadBodyCompositionHistory() once to refresh.
- Wrap HKSampleQuery in withCheckedContinuation (samples, error). On error, skip that type silently.

Validation: body_fat_pct values must be in range 0–100 (they arrive as fraction 0–1 from HealthKit); weight_kg must be positive finite Double before passing to bridge (guard with Double.isFinite && value > 0).

```
VStack(alignment: .leading, spacing: 12) {
  // Header row: icon + title
  HStack {
    Image(systemName: "figure.arms.open")  // or "scalemass" — choose whichever is cleaner
    Text("Body Composition")
      .font(.headline)
    Spacer()
  }

  // Last-logged weight + date (or placeholder when history empty)
  if let last = healthStore.bodyCompositionHistory.last {
    HStack(alignment: .firstTextBaseline, spacing: 4) {
      Text(displayWeight(last))
        .font(.title2.bold())
      Text(weightUnitLabel)
        .font(.subheadline)
        .foregroundStyle(.secondary)
      Spacer()
      Text(last.date)
        .font(.caption)
        .foregroundStyle(.secondary)
    }
    // Body fat % if available
    if let bf = last.bodyFatPct {
      Text(String(format: "Body fat: %.1f%%", bf))
        .font(.caption)
        .foregroundStyle(.secondary)
    }
  } else {
    Text("No data logged yet")
      .font(.subheadline)
      .foregroundStyle(.secondary)
  }

  // Sparkline — hidden when history < 2 points (can't draw a line)
  if healthStore.bodyCompositionHistory.count >= 2 {
    let weightPoints = healthStore.bodyCompositionHistory.compactMap { $0.weightKg }
    if weightPoints.count >= 2 {
      WeightSparklineView(points: weightPoints, tint: .blue)
        .frame(height: 56)
    }
  }

  // Action buttons
  HStack(spacing: 12) {
    Button("Log") {
      showingEntrySheet = true
    }
    .buttonStyle(.bordered)

    Button(isImporting ? "Importing…" : "Import from Health") {
      guard !isImporting else { return }
      isImporting = true
      importError = nil
      Task {
        await healthStore.importBodyCompositionFromHealthKit()
        isImporting = false
      }
    }
    .buttonStyle(.bordered)
    .disabled(isImporting)
  }

  if let err = importError {
    Text(err)
      .font(.caption)
      .foregroundStyle(.red)
  }
}
.sheet(isPresented: $showingEntrySheet) {
  BodyCompositionEntrySheet()
}
```

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-121-01 | Tampering | BodyCompositionEntrySheet weight/BF%/muscle text fields | medium | mitigate | Parse as Double, guard isFinite && > 0 (and < 100 for BF%) before passing to bridge; reject NaN, Inf, negative values |
| T-121-02 | Tampering | importBodyCompositionFromHealthKit BF% fraction | medium | mitigate | Multiply by 100 then guard result in 0–100 range; HealthKit returns 0–1 fraction which would silently persist as near-zero if not converted |
| T-121-03 | Denial of Service | HealthKit authorization denied | low | accept | isHealthDataAvailable() guard prevents crash; UI shows no error on denial (system sheet provides context); function returns gracefully |
| T-121-04 | Information Disclosure | bodyCompositionHistory in @Observable state | low | accept | Local device state only; no network transmission; SQLite is app-sandboxed |
| T-121-05 | Denial of Service | bridge.requestAsync failure during upsert | low | mitigate | Catch errors in upsertBodyComposition(); set errorMessage in entry sheet; do not crash; existing history preserved |
</threat_model>
