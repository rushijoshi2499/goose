<objective>
Replace per-request SQLite Connection::open() with an r2d2 connection pool in the Rust bridge.

Every FFI call from Swift currently opens a fresh SQLite connection (via GooseStore::open), runs work, then closes it. This plan adds a process-global pool so bridge handlers acquire an already-open, already-configured connection — eliminating the file-open and PRAGMA handshake overhead on every call.

Purpose: Eliminate per-request connection open/close overhead across the ~110 bridge call sites. With WAL mode already enabled (confirmed in store/mod.rs:1046), max_size=4 allows concurrent readers without write contention.

Output: 2 files modified. No schema changes. No new Swift changes. No GooseStore API changes.

Decisions implemented: D-01 (r2d2+r2d2_sqlite), D-02 (lazy init, max_size=4 — WAL confirmed), D-03 (OnceLock<Pool> static), D-04 (database_path from bridge call initialises pool).
</objective>
