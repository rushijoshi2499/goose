<objective>
Implement WHOOP BLE reconnect with exponential backoff and its ConnectionView controls (FIX-02), wire the storage compaction bridge method into two Swift call sites (FIX-05, Swift side), and supply the active_device_id arg from the Swift capture write path (FIX-01, Swift side).

Purpose: WHOOP reconnection currently retries immediately with no backoff and no circuit breaker, and there is no UI to observe or control it. This plan introduces a shared `ReconnectBackoff` value type (also consumed by Plan 04 for the HR monitor), refactors `attemptAutomaticReconnect` to drive timed retries through it, and adds the Retry/Stop controls and attempt counter to ConnectionView per the UI-SPEC. It also calls the `storage.compact_raw_evidence` bridge method (added in Plan 01) at launch and after each batch write, and threads the connected peripheral's UUID into the capture import call (consumed by the Rust field added in Plan 02).

Output: new GooseBLEReconnect.swift, refactored WHOOP reconnect path, ConnectionView UI additions, launch + per-write compaction, and active_device_id propagation.

Depends on Plan 01 (storage.compact_raw_evidence bridge method) and Plan 02 (active_device_id Rust field). autonomous: false — contains a human-verify checkpoint for the BLE reconnect UI (no XCTest target; FIX-02 manual verification per VALIDATION.md).
</objective>
