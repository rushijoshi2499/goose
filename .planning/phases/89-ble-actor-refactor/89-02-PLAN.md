<task type="auto">
  <name>Task 1: Create BLESessionCoordinator actor</name>
  <files>GooseSwift/BLESessionCoordinator.swift</files>
  <action>
    Create GooseSwift/BLESessionCoordinator.swift (per D2).

    BLESessionCoordinator is a Swift actor. It holds a reference to the concrete
    CoreBluetoothBLETransport and delegates session lifecycle methods through the actor
    isolation boundary. It does NOT re-implement or replace any BLE logic — all delegation
    is a single pass-through call.

    The actor must expose:

    Stored property (not exposed to callers; used internally and by GooseAppModel init):
    - let transport: CoreBluetoothBLETransport

    Computed property for protocol access (GooseAppModel stores this as `any BLETransport`):
    - var asTransport: any BLETransport { transport }

    Session lifecycle methods (the "thin wrapper" surface — per D2):
    - func connect(startCentral: Bool) — calls transport.requestBluetooth() if startCentral;
      semantics: initiates the BLE central manager lifecycle
    - func startScan() — delegates to transport.startScan()
    - func startScan(reason: String, clearDiscovered: Bool) — delegates to
      transport.startScan(reason:clearDiscovered:)
    - func stopScan() — delegates to transport.stopScan()
    - func stopScan(reason: String) — delegates to transport.stopScan(reason:)
    - func reconnect() — delegates to transport.reconnectRemembered()
    - func disconnect() — delegates to transport.stopScan(); records disconnection via
      transport.record(source: "session", title: "disconnect")

    State queries (nonisolated computed properties that read from the transport — safe because
    CoreBluetoothBLETransport properties are guarded by its own NSLock/queue model):
    - nonisolated var connectionState: String { transport.connectionState }
    - nonisolated var bluetoothState: String { transport.bluetoothState }
    - nonisolated var isScanning: Bool { transport.isScanning }

    Initialiser:
    - init(startCentral: Bool) — creates CoreBluetoothBLETransport(startCentral: startCentral),
      stores it as transport

    Swift 6 strict concurrency notes:
    - The actor isolation means callers must await session lifecycle methods.
    - GooseAppModel calls these from @MainActor context using Task { await coordinator.startScan() }.
    - nonisolated computed properties on the actor are safe because CoreBluetoothBLETransport
      uses NSLock and dedicated DispatchQueues (not actor isolation) for its internal state.
    - Do NOT mark any method @MainActor — the actor boundary is sufficient.

    File header comment: "BLESessionCoordinator — actor thin wrapper around
    CoreBluetoothBLETransport for session lifecycle (connect/disconnect/state). Does not
    replace CoreBluetoothBLETransport internals. See BLETransport.swift for the protocol."
