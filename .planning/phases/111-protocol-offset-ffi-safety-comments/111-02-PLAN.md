# Plan 111-02: FFI Safety Contract Comments

## Objective

Add `// SAFETY:` inline blocks at the two `unsafe extern "C"` FFI entry points:
1. `goose_bridge_handle_json` in `bridge/mod.rs` (C FFI, called from Swift/iOS)
2. `Java_com_goose_app_bridge_GooseBridge_handle` in `android_jni.rs` (JNI, called from Android/Kotlin)

Comments document the caller contract in Rust Nomicon `// SAFETY:` convention. Zero logic changes. `cargo test --locked` passes unchanged.

## Context

Both functions are `pub unsafe extern "C"`. Rust's `// SAFETY:` convention (Nomicon §unsafe-code-guidelines) requires a comment at every `unsafe` block or `unsafe fn` body explaining what invariants the caller must uphold.

- `goose_bridge_handle_json` already has extensive `///` doc comments above it, plus an inline comment on `catch_unwind`. It is **missing** the `// SAFETY:` block at the function body entry.
- `Java_com_goose_app_bridge_GooseBridge_handle` has a brief `/// # Safety` doc comment but it is too terse and **missing** the inline `// SAFETY:` block inside the body.

Both bodies contain `unsafe { CStr::from_ptr(...) }` — those inner unsafe blocks also need `// SAFETY:` comments explaining pointer validity.

**Format (D-03/D-04/D-05):**
```rust
// SAFETY: <pointer/aliasing/lifetime contract>
//   <additional context about caller obligations>
```

### Task 2: Expand `// SAFETY:` block in `Java_com_goose_app_bridge_GooseBridge_handle`

<read_first>
- Rust/core/src/android_jni.rs (full file — 90 lines)
</read_first>

<action>
In `android_jni.rs`, replace the existing brief `/// # Safety` doc comment with a more complete version, and add an inline `// SAFETY:` block at the function body entry.

**Replace the existing doc comment** (`/// # Safety\n/// JNI contract: env and _class are valid for the duration of the call.`) with:
```rust
/// # Safety
///
/// Called by the JVM on a JNI thread. The caller (JVM) guarantees:
/// - `env` is a valid `JNIEnv` pointer for the duration of this call; do not store it beyond the frame.
/// - `_class` is a local JNI class reference valid within this stack frame.
/// - `request` is a local JNI String reference; the JVM will not concurrently mutate it.
///
/// Delegates to `goose_bridge_handle_json`; see that function's SAFETY note for the C FFI contract.
```

**Add inline block at function body entry** (after the opening brace, before `env.get_string`):
```rust
// SAFETY: called by JVM on a JNI thread. `env` is valid for this call's duration,
//   `_class` is a local class ref, and `request` is a local JNI String ref — all
//   guaranteed by the JNI specification. `goose_bridge_handle_json` is called with
//   a CString we construct here; its SAFETY contract is satisfied (non-null, valid UTF-8,
//   duration bounded to this stack frame).
```

**Add `// SAFETY:` on the inner `CStr::from_ptr(response_ptr)`** call:
```rust
// SAFETY: response_ptr is non-null (checked above) and points to a Rust CString
//   allocated by goose_bridge_handle_json; valid until we call goose_bridge_free_string below.
```
</action>
