    Add two new methods to HeartRateSeriesStore immediately after the existing samples(from:to:) method.

    Method 1: func decimatedSamples(from start: Date, to end: Date, maxCount: Int = 500) -> [HeartRateSamplePoint]
    - Call let raw = samples(from: start, to: end) — this acquires and releases stateLock internally. Do NOT acquire stateLock in this method (NSLock is non-reentrant; double-lock on same thread deadlocks).
    - guard raw.count > maxCount else { return raw }
    - let stride = max(1, raw.count / maxCount)
    - var result: [HeartRateSamplePoint] = []
    - result.reserveCapacity(raw.count / stride * 3)
    - Stride loop: var i = 0; while i < raw.count — for each window raw[i..<min(i+stride, raw.count)]:
      - Append raw[i] (first sample in window)
      - If window.max(by: { $0.bpm < $1.bpm })?.id != raw[i].id, append maxSample
      - If window.min(by: { $0.bpm < $1.bpm })?.id != raw[i].id and minSample.id != result.last?.id, append minSample
      - i += stride
    - return result.sorted { $0.capturedAt < $1.capturedAt }

    Method 2: func decimatedSamples(forDayContaining date: Date = Date(), calendar: Calendar = .current, maxCount: Int = 500) -> [HeartRateSamplePoint]
    - Compute dayStart = calendar.startOfDay(for: date)
    - Compute dayEnd = calendar.date(byAdding: .day, value: 1, to: dayStart) ?? dayStart.addingTimeInterval(86400)
    - return decimatedSamples(from: dayStart, to: dayEnd, maxCount: maxCount)

    HeartRateSamplePoint.id is deterministic (confirmed: "\(milliseconds).\(bpm).\(source)"). The != id guards correctly deduplicate within window.
  </action>
