<objective>
Fix two export OOM root causes: (1) the in-memory validation manifest dict is passed by value through multiple bridge calls instead of being released after disk write; (2) validate() is called redundantly from a caller site in addition to the internal call inside createBundle(). Per D-05, D-06, D-07, D-09.

Purpose: Prevent iOS from OOM-killing the app on databases > 100 MB during export by releasing the large manifest dict after it is written to disk, then passing only the file path string to downstream bridge calls.

Output:
- GooseSwift/MoreDataStore.swift — performRawExport uses manifestURL.path for runbook call; no redundant validate() at call site
- Rust/core/src/bridge/debug.rs — LocalHealthValidationManifestRunbookArgs accepts manifest_path; bridge reads JSON from disk
</objective>
