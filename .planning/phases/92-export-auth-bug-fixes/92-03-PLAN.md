<objective>
Fix the WHOOP 5.0 auth stuck state: after the second insufficientAuthentication failure (retry exhausted), the app currently only updates a connectionState string and logs an error. Users have no actionable recovery path. Add a SwiftUI .alert that surfaces the failure and offers Reconnect WHOOP (disconnect + forget) and Cancel (halt loop, keep device) actions. Per D-01, D-02, D-03, D-04.

Purpose: Users stuck in auth failure can recover by tapping "Reconnect WHOOP", which clears the remembered device and returns them to the scan flow. The retry loop is halted immediately — no further authRetryPending triggers.

- GooseSwift/CoreBluetoothBLETransport+PeripheralDelegate.swift — sets authExhausted = true at both retry-exhausted paths; clears it on successful write and on disconnect
- GooseSwift/ConnectionView.swift — .alert bound to ble.authExhausted with Reconnect / Cancel actions
</objective>
